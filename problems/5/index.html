第5問: TODOリスト（その2）

要件

  1. 第4問で作成したTODOに機能を追加
  2. 第4問では個々のTODOはただの文字列だったが、他の属性も入力できるようにする
  3. 追加する属性は好きに決めて良い（複数でも可）
    • ラベル
    • 期限
    • 担当者
    • などなど
  4. 個々のTODOはハッシュで表現する

「ハッシュで表現する」とは

第4問のときには「文字列」としてTODOを表現（=プログラム内でどのような形のデータとして保持するか）していた。この方式では単一の値しか保持できないので、今回のように複数の属性を保持したいときに困ってします。

そこで、個々のTODOを「ハッシュ」で保持する。

ハッシュでデータを保持する際には、具体的に保持する属性の種類と名前を先に設計する。例えば、

  • title: TODOのタイトル
  • label: ラベル

とする。このとき、プログラム内では、例えば、 

todo = {
  title: '日報作成',
  label: 'daily'
}

のように表現する。

出力例

※ 白文字がプログラムが出力したもの、黃文字がユーザが入力したものです。

gist

  • 渡邊
  • 西岡
  • 天野
  • 井原
